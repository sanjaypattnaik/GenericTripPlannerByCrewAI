## AI-Powered Trip Planner Assistant - Requirements

### Project Overview

Build an AI-powered Trip Planner Assistant as a web application using Python and Streamlit. The application should allow users to enter travel details and receive a structured travel plan generated by a CrewAI-based multi-agent system.

The goal is to combine a simple user interface with a multi-agent backend that can research a destination, prepare a local guide, and generate a complete travel plan.

### Required User Inputs

The application must provide the following mandatory input fields:

1. From City
2. Destination City
3. Departure Date
4. Return Date
5. Interests (for example: sightseeing, food, adventure)

These inputs should be collected through the Streamlit interface and passed to the CrewAI workflow.

### Expected Output Reports

After the user submits the travel details, the AI multi-agent system should generate the following outputs:

1. City Report
2. Guide Report
3. Travel Plan

Each output should be clear, structured, and useful for trip planning.

### Multi-Agent System Design

The solution should use CrewAI to implement a multi-agent workflow.

The system should include the following agents:

1. Researcher Agent
    Role: Collect destination-level information and local insights
    Suggested name: City Local Guide Agent

2. Guide Expert Agent
    Role: Build a practical destination guide based on researched information
    Suggested name: Travel Guide Expert

3. Trip Planner Agent
    Role: Create the final end-to-end travel itinerary
    Suggested name: Trip Plan Expert

Each agent should have a clearly defined responsibility so that the workflow remains modular and maintainable.

### Agent Workflow

The agents should run in sequence:

1. Researcher Agent -> prepares destination research
2. Guide Expert Agent -> expands the research into a usable guide
3. Trip Planner Agent -> creates the final travel plan

The output from one agent should be used as input for the next agent in the workflow.

### Task Definition

The following tasks should be implemented in the CrewAI workflow:

1. Location Task
    Assigned to: Researcher Agent
    Output file: city_report.md

2. Guide Task
    Assigned to: Guide Expert Agent
    Output file: guide_report.md

3. Planner Task
    Assigned to: Trip Planner Agent
    Output file: travel_plan.md

### Functional Goal

The final system should accept user travel preferences through the Streamlit interface, coordinate the CrewAI agents sequentially, and generate well-structured travel documents for the selected destination.

### Implementation Notes

To keep the code organized and readable, each task should be implemented as a separate function.

Suggested implementation approach:

1. Create agent definitions for the Researcher Agent, Guide Expert Agent, and Trip Planner Agent.
2. Create task functions for the Location Task, Guide Task, and Planner Task.
3. Import Crew and Process from CrewAI.
4. Build the crew by assigning the agents and tasks in the required sequence.
5. Run the workflow from a final function that executes the crew process.
6. Return the final result and save the generated reports to their corresponding markdown files.

### Expected Technical Outcome

The final implementation should:

1. Accept user input from Streamlit
2. Pass the input into a CrewAI multi-agent workflow
3. Execute the tasks in sequence
4. Generate the final result from the crew process
5. Save the outputs as:
   city_report.md
   guide_report.md
   travel_plan.md
